# Plan Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 落地 Plan 模式：Pat 澄清 → 路径/规格 → OK 确认后关 Plan 并静默下游（仅 Plan→Alex；Plan∧Team→Mike→Alex）；Composer 双开关 + chip；AC5 双保险。

**Architecture:** `resolveTurnKind` 路由；新 `beginPlanTurn`（仅 Pat + `confirm_requirement`）；确认后 SSE `done.nextTurn` + 客户端 `silentHandoff`；下游复用现有 Engineer/Team，注入 `confirmedRequirement`。Plan 与 Team 编排不焊进同一 `runTurn`。

**Tech Stack:** TypeScript、pnpm workspace、vitest、Next.js SSE、现有 shadcn Composer。

**Spec:** `docs/superpowers/specs/2026-07-19-plan-mode-design.md`

## Global Constraints

- 沟通与用户可见文案：简体中文。
- 依赖：`web → application → agent-runtime → agents|llm`；`application → workspace|preview`；Agent 不直碰 `data/**`；禁止 TS 硬编码长 Prompt。
- Plan 澄清不挂载 `write_file`；确认前双保险拒绝 write/enqueue。
- 确认后 **`planEnabled=false`**；静默下游 **不** append handoff user 消息。
- 不做：记忆、权限治理、规格富 UI、Playbook 大改视觉、planner agent。
- **未经用户要求不要 git commit**（下文 commit 步骤一律跳过）。
- 外科手术式改动；测试优先。

```ts
// resolve-turn-kind.ts
export function resolveTurnKind(p: {
  planEnabled: boolean;
  teamEnabled: boolean;
  planConfirmed: boolean;
}): TurnKind {
  if (p.planEnabled && !p.planConfirmed) return "plan_clarify";
  if (p.teamEnabled) return "team";
  return "engineer";
}
```

```ts
confirmRequirement(summary: string) {
  const s = summary.trim();
  if (!s) return { ok: false, error: "摘要不能为空" };
  const proj = deps.workspace.getProject(input.projectId);
  if (!proj) return { ok: false, error: "项目不存在" };
  const nextTurn = proj.teamEnabled ? "team" : "engineer";
  deps.workspace.updateProjectMeta(input.projectId, {
    planConfirmed: true,
    confirmedRequirement: s,
    planEnabled: false,
  });
  // 在闭包记录 confirmedThisTurn=true, nextTurnForDone=nextTurn
  return { ok: true };
}
```
