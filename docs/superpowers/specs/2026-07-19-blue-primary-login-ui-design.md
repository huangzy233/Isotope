# 设计：蓝主色换肤 + 克制分栏登录页

- 日期：2026-07-19
- 状态：已批准（对话确认）
- 范围：全局 Design Token 主色改为 `#2563EB`；登录页改为左右分栏（文案仅反映已有能力）
- 前置决策：方案 A（蓝主色）+ 范围 B（换肤 + 登录）+ 登录方案 2（克制分栏）

## 1. 目标

1. 将产品主色从近黑 Neutral Tool 调整为图例中的蓝主色 `#2563EB`，使主按钮、焦点环、活跃态与参考图一致。
2. 登录页改为左右分栏：左侧展示品牌与已落地能力，右侧保留演示账号登录表单。
3. 文案诚实：不出现未实现能力（实时多人协作、企业数据安全、开放注册、忘记密码等）。

## 2. 非目标

- 不改首页 / 工作台的页面结构与信息架构（仅随 token 自动变色）
- 不做营销落地页、3D 装饰图、紫粉渐变、glow
- 不增加「记住我」「忘记密码」「立即注册」「一键演示账号」等未实现交互
- 不引入新 UI 库；仍用 shadcn + Tailwind 语义 class
- 不改鉴权逻辑（演示账号、会话 cookie、middleware 门禁保持现状）

## 3. Design Tokens

### 3.1 变更

| Token | 旧值 | 新值 | 用途 |
|-------|------|------|------|
| `--primary` | `#0F172A`（近黑） | `#2563EB` | 主按钮、活跃态 |
| `--primary-foreground` | `#FFFFFF` | `#FFFFFF`（不变） | 主按钮文字 |
| `--ring` | 与旧 primary 一致 | `#2563EB` | focus ring |

其余 token（`--background`、`--foreground`、`--border`、`--muted*`、`--success`、`--warning`、`--destructive`、`--radius`、`--shadow-soft`）保持现有 slate 灰阶与状态色。

### 3.2 实现落点

- `apps/web/app/globals.css`：更新 CSS 变量
- `docs/ui/design-system.md`：将「Neutral Tool / 近黑 primary」改为「蓝主色 `#2563EB`」；保留「禁止紫粉渐变 / glow / 主按钮 rounded-full / Primary 大面积铺底」等约束
- 组件继续使用语义 class（`bg-primary`、`ring-ring` 等），不在页面硬编码 `#2563EB`

### 3.3 规则补充

- Primary 仍**不**用于全屏 Hero 大面积铺底；登录页左侧底色约定为 `bg-primary/5`（无渐变、无 glow），不得做成高饱和蓝墙
- 品牌仅用文字「Isotope」（现网无独立 logo 资源；本轮不新增六边形等图标资产）
- 链接 / 选中态可用 primary；禁止用高饱和蓝紫当装饰墙

## 4. 登录页

### 4.1 布局

```text
md+（≥768px）:
┌─────────────────────────────┬──────────────────────────┐
│  左：品牌区 ~50%             │  右：表单区 ~50%           │
│  文字品牌 Isotope           │  居中 Card（max-w-md）     │
│  标题 + 一句副文案           │  品牌名 + 副文案            │
│  3 条能力（线型图标+标题+说明）│  用户名 / 密码 / 登录       │
│  底：`bg-primary/5`（无渐变） │  错误靠近表单               │
└─────────────────────────────┴──────────────────────────┘

窄屏（<768px）:
┌──────────────────────────┐
│  仅右表单区（全宽居中）     │
│  左侧品牌区隐藏            │
└──────────────────────────┘
```

### 4.2 文案（定稿）

**左侧**

| 元素 | 文案 |
|------|------|
| 品牌 | Isotope |
| 标题 | 智能应用生成平台 |
| 副文案 | 用 AI 对话构建应用，并实时预览 |
| 能力 1 标题 | 对话驱动生成 |
| 能力 1 说明 | 自然语言描述需求，智能体生成并迭代 |
| 能力 2 标题 | 实时预览 |
| 能力 2 说明 | 改码后自动构建，App Viewer 即时更新 |
| 能力 3 标题 | Plan / Team 模式 |
| 能力 3 说明 | 需求澄清与多智能体编排可按需开关 |

**右侧**

| 元素 | 文案 |
|------|------|
| 标题 | Isotope |
| 副文案 | 使用演示账号登录以继续 |
| 字段 | 用户名 / 密码（placeholder：`demo` / `••••••••`） |
| 主按钮 | 登录 → 提交中 `登录中…` |
| 错误 | 沿用现有接口错误文案 |

### 4.3 明确不做

- 实时协作、数据安全可靠、灵活扩展（图例有、产品未就绪或表述不当）
- 记住我、忘记密码、或、使用演示账号登录快捷按钮、立即注册
- 左侧抽象 3D 数据可视化装饰图

### 4.4 组件与文件

- 页面：`apps/web/app/(public)/login/page.tsx` — 分栏壳
- 表单：`apps/web/components/login-form.tsx` — 逻辑不变；如需可微调 class 以适配右侧 Card
- 原子：继续用现有 `Card` / `Button` / `Input` / `Label`；能力列表可用简单 `ul` + 线型图标（lucide，与正文对齐，约 16px）
- 禁止新建 `.module.css` 或页面内大段自定义皮肤

### 4.5 状态与无障碍

- Empty / Loading / Error 行为与现网一致
- 焦点环使用新 `--ring`（蓝）
- 左侧能力列表为装饰性说明，不抢主任务；窄屏隐藏不影响登录可达性
