# 设计：工作台沉浸模式（应用查看器 / 编辑器）

- 日期：2026-07-19
- 状态：已批准（对话确认）
- 范围：工作台右侧主区沉浸布局；不改 preview 构建、侧栏、领域包

## 1. 目标

让用户在工作台内用更大视口体验生成结果或阅读源码：

1. 一键进入沉浸：隐藏左侧对话区与分栏拖拽条，右侧主区铺满工作台内容区宽度。
2. 「应用查看器」与「编辑器」均可沉浸；沉浸中仍可 Tabs 切换两种模式。
3. 顶栏保留现有控件（Tabs；预览时的 StatusBadge + 刷新），并提供进入/退出沉浸按钮。
4. 退出：点「退出沉浸」或按 Esc；状态仅会话内，不持久化。

## 2. 非目标

- 浏览器 Fullscreen API / 系统级全屏
- 新标签页打开预览
- 记住沉浸偏好（localStorage / 跨刷新）
- 隐藏或改动 `AppSidebar` / 全局壳
- 拆分 `ViewerChrome` / 大范围重构 `workbench-shell`
- 改动 preview 构建、iframe 代理、编辑器读文件逻辑

## 3. 已确认的产品决策

| 决策 | 选择 |
|------|------|
| 形态 | 工作台沉浸（藏对话，主区铺满），非浏览器全屏 |
| 顶栏 | 保留完整顶栏（Tabs + 预览状态/刷新） |
| 模式 | preview 与 editor 共用同一 `immersive` 状态 |
| 退出 | 顶栏按钮 + Esc |
| 持久化 | 不写 localStorage；刷新后回到双栏 |
| 侧栏 | 项目侧栏行为不变 |

## 4. 交互细则

### 4.1 进入

- 入口：右侧主区顶栏按钮（文案或图标与现有 `Button` `size="sm"` / `variant="outline"` 或 `ghost` 对齐；可用 lucide `Maximize2`）。
- 任意 `viewerMode`（`preview` | `editor`）下均可进入。
- 进入后：对话 `section` 与垂直分隔条不渲染（或 `hidden`）；右侧 `section` 占满主区。

### 4.2 沉浸中

- Tabs 仍可在「应用查看器」与「编辑器」间切换；不退出沉浸。
- 预览 Ready / Building 时仍显示 StatusBadge 与「刷新」。
- 顶栏进入按钮变为「退出沉浸」（可用 `Minimize2`）。
- `chatPct` 保持内存中的值，退出后分栏比例不变。

### 4.3 退出

- 点击「退出沉浸」→ `immersive = false`，恢复对话区与分隔条。
- 按 Esc → 同上。非沉浸时 Esc 无副作用。
- 不拦截其他快捷键；沉浸时对话 Composer 已隐藏，无「输入框 Esc」冲突。编辑器内按 Esc 也退出沉浸（接受为明确产品选择）。

## 5. 实现落点

| 层 | 变更 |
|----|------|
| `apps/web/components/workbench-shell.tsx` | `immersive` state；条件渲染对话/分隔条；顶栏按钮；`keydown` Esc |
| 其他包 / API | 无 |

可选：图标来自已有 `lucide-react` 依赖，与侧栏 `PanelLeft` 风格一致。

## 6. 验收标准

1. 预览 Ready 时点沉浸 → 对话与分隔条消失，iframe 区域变宽。
2. 沉浸中切到编辑器 → 仍沉浸，编辑器铺满主区。
3. 点「退出沉浸」或 Esc → 恢复双栏，且 `chatPct` 与进入前一致。
4. 非沉浸时按 Esc → 布局无变化。
5. 窄屏（对话与预览上下堆叠）下沉浸仍隐藏对话区，主区占满可用高度/宽度。

## 7. 测试建议

- 以组件/手工验收为主（布局与键盘）；无后端契约变更。
- 若已有 workbench 相关测试工具，可为「Esc 仅在 immersive 时置 false」抽极小纯函数或直接手工点验。
