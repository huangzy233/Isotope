# Page Blueprints

AI 按页搭架子。先读对应 Blueprint，再写代码。

宽度 / 字号等 **数值约定** 见 [design-system](./design-system.md)。  
Composer / Agent / Viewer **状态机** 见 [ai-surfaces](./ai-surfaces.md)。

本轮覆盖：Login · Home · Workspace · App Header。  
Settings 等未立项页面：**不要发明 Blueprint**，等有产品需求再补。

---

## Blueprint 模板（每页相同字段）

每页必须能回答：目标、主任务、必须/可选模块、ASCII、宽度密度、首屏/第二屏、状态、响应式、为何、反例。

## Login `/login`

### 页面目标

让已授权用户完成登录，进入应用壳。

### 主要任务 / Primary Action

提交凭证 → 进入首页。Primary Action = 「登录」按钮。

### 信息架构

| 必须 | 可选 |
|------|------|
| 品牌名（Page Title 级） | 一句欢迎副文案 |
| 表单（Label + Input ×N） | 「忘记密码」等（若产品有） |
| 主按钮 | — |
| 错误展示（靠近表单） | — |

### 推荐布局 + ASCII

宽度：Auth（`max-w-md` 居中）。密度：中等；垂直勿撑满成营销页。

```text
┌──────────────────────────────────────────┐
│              (viewport)                  │
│                                          │
│         ┌────────────────────┐           │
│         │  Isotope           │           │
│         │  欢迎回来           │           │
│         │                    │           │
│         │  Email             │           │
│         │  [..............]  │           │
│         │  Password          │           │
│         │  [..............]  │           │
│         │  [    登录      ]  │           │
│         │  (error near form) │           │
│         └────────────────────┘           │
│                                          │
└──────────────────────────────────────────┘
```

### 首屏 / 第二屏

- **首屏：** 品牌 + 完整表单 + 主按钮（无第二屏内容）。
- **不要：** uppercase marketing eyebrow、大英雄图、侧栏卖点。

### 状态

- Empty：表单空闲。
- Loading：提交中按钮 disabled / 文案切换。
- Error：文案靠近字段或表单顶，勿全屏大红。

### 响应式

窄屏仍居中窄栏；保证触控可点，不横向撑破。

### 为什么这样布局

登录只有一个任务。窄栏 + Card 包裹降低噪音，品牌用标题级即可，避免做成 Landing。

### 本页反例

- 紫渐变全屏背景 + 居中大口号
- 表单两侧放功能介绍卡片
- 主按钮胶囊全圆角 + glow

## Home `/`

### 页面目标

发起新构建，并回看已有项目。

### 主要任务 / Primary Action

Composer：填写需求 → 选模式 → **开始**。  
次要：进入已有项目。

### 信息架构

| 必须 | 可选 |
|------|------|
| Composer（主） | 模式差异一句 Secondary 说明 |
| 「我的项目」Section + 列表或 EmptyState | 项目行元数据（时间等） |
| App Header（壳层） | — |

### 推荐布局 + ASCII

宽度：App Content（`max-w-2xl`～`max-w-3xl`）。密度：偏高；区块间距 16–24，避免巨大留白。

```text
┌─────────────────────────────────────────────┐
│ Header: Isotope · user · 退出               │
├─────────────────────────────────────────────┤
│         (max-w-3xl centered)                │
│  ┌───────────────────────────────────────┐  │
│  │ Composer                              │  │
│  │ [需求 textarea                    ]   │  │
│  │ [Engineer|Team]          [ 开始 ]     │  │
│  └───────────────────────────────────────┘  │
│                                             │
│  我的项目                                   │
│  ┌───────────────────────────────────────┐  │
│  │ project-a · meta                      │  │
│  │ project-b · meta                      │  │
│  └───────────────────────────────────────┘  │
│  (或 EmptyState)                            │
└─────────────────────────────────────────────┘
```

### 首屏 / 第二屏

- **首屏：** Composer 完整可见（视觉中心）；其下至少露出项目 Section 标题或首行。
- **第二屏：** 更长的项目列表。
- Trace / Cost / 技术指标：**不进**首屏。

### 状态

- Empty 项目：EmptyState（标题 + 一句说明；可指向上方 Composer）。
- Composer submitting：主按钮 loading / disabled。
- 空输入：实现阶段应拦截（规范要求）。

### 响应式

内容栏可随屏变窄；保持单栏，不改成双栏仪表盘。

### 为什么这样布局

一屏一主任务：发起。项目列表是支持模块，权重低于 Composer。中等内容宽保证扫读，全宽会显得空。

### 本页反例

- 只有一句 Slogan + 巨大留白，无第二内容模块
- Composer 做成营销大卡片（过重阴影 / 过大 padding）
- 首屏堆统计条、徽章、多 CTA

## Workspace `/projects/[id]`

### 页面目标

在项目上下文中对话，并对照 App Viewer 预览结果。

### 主要任务 / Primary Action

发送消息（左栏 Composer）；理解构建/预览状态（右栏）。  
双主叙事：对话流 + Viewer，同一工作台。

### 信息架构

| 必须 | 可选 |
|------|------|
| 左：PanelHeader（对话 + Agent 状态） | Tool 详情展开 |
| 左：消息区 | Trace / Cost 折叠 |
| 左：Composer | 刷新等 Viewer 次要操作 |
| 右：ViewerChrome + 预览区 | — |
| 项目名 / id（Metadata，可读） | — |
| App Header | — |

### 推荐布局 + ASCII

宽度：Workbench **全宽**。密度：高（`sm`/`xs` 为主）。Desktop ≥1280px 双栏。

```text
┌──────────────────────────────────────────────────────────┐
│ Header: Isotope · project meta · user · 退出             │
├────────────────────────────┬─────────────────────────────┤
│ PanelHeader 对话 · status  │ ViewerChrome · status       │
├────────────────────────────┼─────────────────────────────┤
│ messages...                │                             │
│ messages...                │   App Viewer / preview      │
│                            │                             │
├────────────────────────────┤                             │
│ Composer                   │                             │
│ [textarea]        [发送]   │                             │
└────────────────────────────┴─────────────────────────────┘
```

### 首屏 / 第二屏

- **首屏：** 双栏结构完整；左可见最近消息或 Empty；右可见 Viewer 状态（Idle/Building/Ready/Failed）。
- **滚动：** 消息历史上卷；Trace 等不抢首屏。

### 状态

- 消息 Empty：说明可开始对话。
- Viewer Idle / Building / Ready / Failed：见 ai-surfaces；Failed 须文案 + 重试。
- Agent / Streaming：StatusBadge 可辨。

### 响应式

| 断点 | 要求 |
|------|------|
| ≥1280px | 双栏完整 |
| Tablet | 上下堆叠或抽屉式聊天 |
| Mobile | 不崩；Composer ↔ Viewer 可切换即可 |

Desktop 优先；Mobile 底线是可用、不坏。

### 为什么这样布局

对话是主叙事，预览是结果面；全宽双栏是 IDE/工具工作台惯例。窄内容宽会浪费预览，也会挤消息。

### 本页反例

- 内容区再套 `max-w-page` 居中
- 彩色聊天气泡墙
- 每个 Tool 闪亮动画刷屏
- 把 Trace/Cost 做成首屏大卡片

## App Header（跨页壳）

### 页面目标

提供品牌锚点、用户上下文、退出；在项目页补充可读的项目上下文。

### 主要任务 / Primary Action

无页级主 CTA。品牌可回首页；退出为明确动作。

### 信息架构

| 必须 | 可选 |
|------|------|
| 产品名 | 当前模式（轻量） |
| 用户上下文 | 构建状态（轻量） |
| 退出 | 项目名 / id（项目页） |

### 推荐布局 + ASCII

全宽顶栏；高度紧凑（约面板头量级）。

```text
┌──────────────────────────────────────────────────────────┐
│ Isotope          (project meta)        user · 退出       │
└──────────────────────────────────────────────────────────┘
```

### 首屏

Header 始终在首屏顶部；不占垂直英雄区。

### 为什么这样布局

壳层导航服务定位与退出，不是营销导航。信息保持 Metadata / 轻量级，避免顶栏变成仪表盘。

### 本页反例

- 顶栏放大 Logo + 多组营销链接
- 顶栏堆满徽章与统计
- 项目页只显示裸 id 无可读名称
