# Design Principles

只答 **为什么**。不含 Token、样式工具、组件名。

## 1. 产品是谁

Isotope 是 AI 驱动的应用构建工具：用户描述需求 → Agent 协作 → 预览结果。

**目标用户：** 开发者与产品人 — 需要快速发起、扫读状态、对照预览，而不是浏览营销故事。

**主路径：** 登录 → 首页发起 → 工作台（对话 + App Viewer）。

## 2. 目标气质：Neutral Tool

关键词：Modern SaaS · Minimal · Professional · Clean · High information density · Developer friendly。

选定气质：**Neutral Tool** — 中性灰阶承载界面；主操作近黑；彩色只服务语义状态。

最终标准：界面可展示给用户与投资人，而不是「开发者占位页」。

## 3. 为什么 Neutral Tool

工具 UI 的主角是 **内容与任务**，不是品牌装饰。

- 灰阶默认 → 可信、可扫读、少情绪噪音。
- 近黑主操作 → 清晰「这是该点的」；不靠高饱和色抢注意力。
- 彩色仅状态 → 绿 / 琥珀 / 红有语义；多色墙会毁掉层级。

## 4. 为什么高信息密度

工作台是主战场：消息、状态、预览同时存在。大片空洞会让产品看起来像 Demo 或 Landing。

**密度 ≠ 拥挤：** 靠字重、字号、弱边框、分组留白建立层级；不是把控件塞满。

## 5. 为什么少颜色、少装饰

- 每多一种装饰色，就多一个竞争焦点。
- 渐变、光晕、粒子、糖果圆角 = 「Demo 脸」——像模板站，不像持续使用的产品。
- 分层优先用留白与字重，而不是粗边框或重阴影。

## 6. 为什么不做 Marketing / Landing

本产品壳层是 **工具页**，不是获客页。

| 工具页 | Marketing |
|--------|-----------|
| 一屏一主任务 | 英雄口号 + 多 CTA |
| 输入 / 列表 / 预览为中心 | 卖点卡片墙 |
| 元数据弱化 | 统计条、徽章贴纸 |
| 品牌克制 | uppercase eyebrow、大展示标题压过内容 |

不做开放营销落地页专项；不做「投资人演示风」第一屏。

## 7. 为什么参考 Linear 等

参考：Linear、Cursor、Claude、Vercel、Notion、Stripe Dashboard。

**取规律，不取皮肤：**

- 工具感、信息密度、弱边框、近黑主操作、清晰层级。
- **不**复制某一家的色板、插画或侧栏皮肤。

细节规律见 [inspirations.md](./inspirations.md)。

## 8. 六条硬原则

1. **工具感优先于营销感** — 少英雄区、少装饰图形、少口号墙。
2. **中性默认，彩色仅语义** — 几乎全灰阶；状态色克制。
3. **高信息密度，仍可读** — 紧凑但标题 / 正文 / 元数据可扫读。
4. **一屏一主任务** — 首页 = 发起构建；工作台 = 对话 + 预览。
5. **状态可见** — 空闲 / 运行 / 流式 / 成功 / 失败必须可辨。
6. **禁止 Demo 脸** — 无紫粉渐变、无光晕、无糖果大圆角、无 emoji 堆砌、无科技风粒子。

## 9. 非目标

- 不引入另一套大型 UI 库替换现有组件源。
- 不做深色模式专项（能力可保留，规范以浅色为准）。
- 不在本文规定后端、LLM 或构建协议。
- 不做 Settings 等未立项页面的布局发明（有需求时再补 Blueprint）。
